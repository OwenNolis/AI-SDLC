# Technische Analyse: Feature-002: Support tickets zoeken en filteren

## 1. Scope

### In scope
- Overzicht van support tickets tonen
- Zoeken op ticketnummer en subject
- Filteren op status
- Filteren op priority
- Sorteren op aanmaakdatum
- Lege resultaten correct tonen

### Out of scope
- Ticket aanmaken
- Ticket bewerken of sluiten
- Paginatie
- Export van resultaten
- Geavanceerde full-text search

## 2. Assumptions
- Er is een bestaande API beschikbaar voor het ophalen van support tickets.
- De API retourneert tickets in een gestructureerd formaat met minimaal de volgende velden: `id`, `ticketNumber`, `subject`, `status`, `priority`, `createdAt`.
- De frontend applicatie is gebouwd met een modern JavaScript framework (bv. React, Vue, Angular).

## 3. Open Questions
- Wat is het exacte pad en de structuur van de API endpoint voor het ophalen van tickets?
- Welke specifieke waarden worden verwacht voor `status` en `priority` filters?
- Hoe wordt de correlatie-ID gegenereerd en doorgegeven aan de backend?

## 4. Domain Model

| Entiteit      | Velden           | Constraints                               |
|---------------|------------------|-------------------------------------------|
| Ticket        | id (UUID)        | Primary Key                               |
|               | ticketNumber (String) | Uniek, Exacte match bij zoeken            |
|               | subject (String) | Gedeeltelijke match bij zoeken            |
|               | status (Enum)    | OPEN, CLOSED (Filterbaar)                 |
|               | priority (Enum)  | LOW, MEDIUM, HIGH (Filterbaar)            |
|               | createdAt (DateTime) | Sorteerbaar (oplopend/aflopend)           |
|               | updatedAt (DateTime) |                                           |
|               | description (String) |                                           |

## 5. API Design

### Gebruikte endpoints
| Method | Path | Wanneer | Request Body | Response Body |
|--------|------|---------|--------------|---------------|
| GET    | `/api/tickets` | Bij laden van `TicketListPage` en na interactie met zoek/filter/sort controls | Query parameters: `searchQuery` (string), `status` (string), `priority` (string), `sortBy` (string, bv. `createdAt_desc`), `sortOrder` (string, bv. `asc` of `desc`) | Array van Ticket objecten |

### Error handling
- **4xx Client Errors**:
    - **400 Bad Request**: Ongeldige filterwaarden. Toon gebruikersvriendelijke foutmelding via `ErrorMessage` component, bv. "Ongeldige filterwaarde voor status."
    - **401 Unauthorized / 403 Forbidden**: Gebruiker is niet geauthenticeerd of geautoriseerd. Toon generieke foutmelding of redirect naar login.
    - **404 Not Found**: Resource niet gevonden. Toon generieke foutmelding.
- **5xx Server Errors**:
    - **500 Internal Server Error**: Generieke serverfout. Toon generieke foutmelding met `correlationId` via `ErrorMessage` component.
- **Netwerk Fouten**:
    - Geen verbinding, timeout, etc. Toon verbindingsfoutmelding via `ErrorMessage` component.

**Standaard Error Formaat (Backend Response):**
```json
{
  "correlationId": "uuid-van-request",
  "message": "Een beschrijvende foutmelding",
  "details": "Optionele gedetailleerde foutinformatie"
}
```

## 6. Backend Design

*   **TicketService**: Verantwoordelijk voor het ophalen en filteren van tickets vanuit de datastore.
    *   `getTickets(filters, sortOptions)`: Methode die de logica bevat voor het toepassen van zoekopdrachten, filters en sortering op de ticketdata.
*   **TicketController**: Exposeert de `/api/tickets` endpoint.
    *   Valideert de inkomende query parameters.
    *   Roept de `TicketService` aan met de juiste parameters.
    *   Formatteert de response, inclusief foutafhandeling.
*   **Data Access Layer**: Verantwoordelijk voor interactie met de database.

## 7. Frontend Design

### Routes
- `/{path}/tickets` — Toont de lijst van support tickets met zoek-, filter- en sorteeropties.

### Componenten
| Component         | Verantwoordelijkheid                                                              |
|-------------------|-----------------------------------------------------------------------------------|
| `TicketListPage`  | Hoofdcomponent voor de ticketoverzichtspagina. Beheert de state van zoek-, filter- en sorteeropties en roept de API aan. |
| `TicketList`      | Rendert de lijst van `TicketListItem` componenten.                                |
| `TicketListItem`  | Toont de details van een individueel ticket.                                      |
| `SearchBar`       | Inputveld voor het zoeken op ticketnummer en subject.                             |
| `StatusFilter`    | Dropdown voor het filteren op ticketstatus (OPEN, CLOSED).                        |
| `PriorityFilter`  | Dropdown voor het filteren op ticketprioriteit (LOW, MEDIUM, HIGH).               |
| `SortOrderSelector` | Selectie voor het sorteren op aanmaakdatum (nieuwste/eerst, oudste/eerst).        |
| `LoadingSpinner`  | Toont een visuele indicator tijdens het laden van data.                           |
| `EmptyState`      | Toont een melding wanneer er geen tickets gevonden worden.                        |
| `ErrorMessage`    | Toont foutmeldingen aan de gebruiker, inclusief correlationId bij serverfouten.   |

### State management
- **`TicketListPage` State**:
    - `searchQuery`: String, beheerd door `SearchBar`.
    - `statusFilter`: String, beheerd door `StatusFilter`.
    - `priorityFilter`: String, beheerd door `PriorityFilter`.
    - `sortOrder`: String (bv. 'createdAt_desc'), beheerd door `SortOrderSelector`.
    - `tickets`: Array van Ticket objecten, opgehaald via API.
    - `loading`: Boolean, geeft aan of data wordt geladen.
    - `error`: Object, bevat foutinformatie indien van toepassing.
- **State Management Tool**: Context API of Redux/Zustand voor centrale state management indien nodig voor complexere applicaties.

| `TicketListPage`  | Hoofdcomponent voor de ticketoverzichtspagina. Beheert de state van zoek-, filter- en sorteeropties en roept de API aan. |
| `TicketList`      | Rendert de lijst van `TicketListItem` componenten.                                |
| `TicketListItem`  | Toont de details van een individueel ticket.                                      |
| `SearchBar`       | Inputveld voor het zoeken op ticketnummer en subject.                             |
| `StatusFilter`    | Dropdown voor het filteren op ticketstatus (OPEN, CLOSED).                        |
| `PriorityFilter`  | Dropdown voor het filteren op ticketprioriteit (LOW, MEDIUM, HIGH).               |
| `SortOrderSelector` | Selectie voor het sorteren op aanmaakdatum (nieuwste/eerst, oudste/eerst).        |
| `LoadingSpinner`  | Toont een visuele indicator tijdens het laden van data.                           |
| `EmptyState`      | Toont een melding wanneer er geen tickets gevonden worden.                        |
| `ErrorMessage`    | Toont foutmeldingen aan de gebruiker, inclusief correlationId bij serverfouten.   |

### State management
- **`TicketListPage` State**:
    - `searchQuery`: String, beheerd door `SearchBar`.
    - `statusFilter`: String, beheerd door `StatusFilter`.
    - `priorityFilter`: String, beheerd door `PriorityFilter`.
    - `sortOrder`: String (bv. 'createdAt_desc'), beheerd door `SortOrderSelector`.
    - `tickets`: Array van Ticket objecten, opgehaald via API.
    - `loading`: Boolean, geeft aan of data wordt geladen.
    - `error`: Object, bevat foutinformatie indien van toepassing.
- **State Management Tool**: Context API of Redux/Zustand voor centrale state management indien nodig voor complexere applicaties.
